# 监管平台 UI 通用指南

这份指南把 Codex skill 中的监管平台页面规则改写成任何 AI 都能读取的说明。

## 设计目标

生成的页面应像监管平台工作台，而不是通用 SaaS、营销页、电商后台或装饰型大屏。

核心气质：

- 克制
- 专业
- 高信息密度
- 适合业务人员反复操作
- 保留原业务流程和字段

## 页面理解

先把需求编译成页面理解：

- 页面名称
- 页面类型
- 业务对象
- 使用角色
- 核心目标
- 关键字段
- 主要操作
- 业务状态
- TODO

页面类型包括：

- list：列表页、筛选、表格、批量操作
- detail：详情页、基础信息、流程记录、关联内容
- dialog：弹窗、确认、表单、反馈
- dashboard：业务看板、指标、趋势、待办
- empty state：无数据、筛选为空、无权限或加载失败
- mixed workflow：多个类型组合的流程页

## 必须复用的规则

1. 使用左侧导航、顶部栏、右侧工作区的后台结构。
2. 表格、筛选、状态、流程记录是主要信息载体。
3. 页面背景使用 `assets/bg.jpg` 和浅色底色，不使用花哨渐变。
4. 功能图标优先来自 `assets/icons/`。
5. 英文和数字字体可使用 `assets/fonts/D-DIN-PRO-*.otf`。
6. 控件结构遵守 Dorami 组件协议。
7. 字段、状态、操作按 `data/field-component-map.json` 映射。
8. 业务对象、操作、状态优先级按 `data/business-semantics.json` 判断。
9. 页面输出前必须按 `compiler/validation-checklist.md` 自检。

## 禁止方向

- 不要做 Hero Banner。
- 不要做营销页。
- 不要做炫酷数据大屏。
- 不要大面积使用装饰渐变、彩色阴影、胶囊按钮。
- 不要为了美观删除业务字段、表格列或操作入口。
- 不要自创与监管平台无关的组件。

## 输出格式建议

让 AI 输出：

1. 页面理解
2. 结构规划
3. 组件映射
4. token 和资源引用说明
5. 页面代码或页面结构
6. 校验清单结果
7. TODO 和信息缺口
